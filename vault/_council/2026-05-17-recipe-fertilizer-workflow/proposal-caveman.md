# Caveman Proposal: One Card Per Formula

## Core idea (1 sentence)

One file per formula. Write what you put in. Write what happened. Stop.

## What user does (physical steps)

**For food recipe:**
1. Cook it.
2. Open the file. Write what you changed. Write how it tasted. One paragraph.
3. Done. Next time, read that file before cooking again.

**For fertilizer:**
1. Mix the batch. Write exactly what you mixed. Exact amounts.
2. Put it on the tree. Write the date and which tree.
3. Wait. Check the tree. Write what you see.
4. Do not change the formula until you write what the current one did.

That is the whole workflow. No slash command needed yet. No system needed yet. Just the file.

## ONE file per formula — what goes in it

**Food file:**
```
Name: [dish name]
Base reference: [where you learned it — video, paper, memory]
Version: v1 / v2 / v3...
What I put in: [exact ingredients, exact amounts]
What I changed from last time: [one line]
How it tasted: [honest 2-3 sentences]
What to try next: [one specific change]
```

**Fertilizer file:**
```
Name: [formula name]
Target: palm tree
Version: v1 / v2 / v3...
What I mixed: [every ingredient, exact weight/ratio]
Applied to: [which trees, how many]
Applied on: [date]
Observed on: [date, what the tree looks like]
Verdict: keep / change / stop
What to change next: [one specific thing only]
```

That is it. Two templates. Nothing else.

## When to test / when to stop

**Food:** Test immediately. If it tastes bad, write why. Change ONE thing for v2. Not three things. One. You need to know what fixed it.

**Fertilizer:** Test on 2-3 trees only. Not the whole field. Watch for 6 weeks minimum before calling it good or bad. Never change more than one ingredient between versions. If trees look sick — STOP. Do not continue testing on more trees. Write what happened. Wait for advice before v2.

**When to call it "ready":** Food — 3 people eat it and say good without you telling them what to fix. Fertilizer — 2 full growth cycles show same or better results, no damage.

**When to stop entirely:** Food — after 5 versions still tastes bad, the base recipe is wrong. Find a new base. Fertilizer — any sign of leaf burn, root damage, or yield drop below baseline. Stop the formula. Do not "push through" to see if it gets better.

## Gut signal: UNEASY

Two domains at once. Different speeds. Different stakes. That is two hunting trips in opposite directions.

Food fails: you eat something bad. One meal.

Fertilizer fails: you kill trees that took years to grow. You lose the season. Real money. Real time you cannot get back.

The danger is not the system. The danger is treating them the same. They are not the same.

Food = fast, cheap, reversible. Fertilizer = slow, expensive, permanent.

Run food first. Get the habit of tracking. Then bring that habit to fertilizer.

Do not start both on day one.

The second danger: the habit dies in week 3. You get busy. You stop writing the file after testing. Then the file is useless. The only thing that makes this work is the 2-minute write after every test. Every single time. No exceptions.

If you skip writing once — you will skip it again. Then you have no memory. Then you start over. That is the real cost.

Start with ONE recipe. Track it perfectly for one month. Then add fertilizer. Then think about slash commands.
