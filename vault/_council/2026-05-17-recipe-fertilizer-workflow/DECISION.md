# Council Decision: Recipe + Fertilizer Workflow

## TL;DR

4 agents debate 4 approaches: full integrated pipeline (Optimist), shared spine with domain gates (Pragmatist), sequential one-domain-at-a-time (Skeptic), and radical minimum one-file (Caveman). The debate is NOT "which template" — it is "which bottleneck": habit formation, knowledge compounding, or fertilizer safety. The answer determines which hybrid to use.

**Pragmatist wins (28pts).**

## Expertise warnings (Strategist)

- **Start a baseline fertilizer log THIS WEEK** — 30 minutes, document current fertilizer/cost/palm condition now. Every untracked season is permanently lost. Do this regardless of which workflow you choose.
- **Recipe + content creation should be linked** — add `content_angle:` field to recipe template if external value (filming, teaching) is a goal. Zero extra cost.
- **Validate fertilizer ROI assumption first** — organic inputs are sometimes MORE expensive per effective NPK than synthetic. Confirm local zero-cost input sources exist before 12-month commitment.

---

## Caveman gut signal

**UNEASY** — caught the thing sophisticated proposals missed: food failure = one bad meal, fertilizer failure = lost season + real money. The gut is right that habit must come before system. The UNEASY signal does NOT contradict the Pragmatist structure — it contradicts launching both domains simultaneously without a proven tracking habit.

---

## Recommendation framework

**IF habit formation is the bottleneck** (no consistent tracking habit yet):
→ Caveman template for 4 weeks (food only), then upgrade to Pragmatist. Do not build the system before the habit proves real.

**IF knowledge compounding is the goal** (recipes + fertilizer connect to content / teaching / external value):
→ Hybrid A: Pragmatist structure + baseline log first + `content_angle:` field + 12-week fertilizer gate

**IF fertilizer safety is the priority** (significant crop at risk):
→ Skeptic's sequencing + Pragmatist structure: food recipes first for 2 months, fertilizer only after 8 tracked recipe iterations, 12-week minimum gate, test on maximum 3 trees before scaling

**IF uncertain:**
→ Answer the 2 questions in Hard Questions below first. Do not build workflow until you know the answer to Q2.

---

## Points of consensus across all 4 agents

1. No slash commands at start — manual workflow first
2. One variable changed per iteration (scientific discipline)
3. Food recipes build habit faster — start there
4. Version files required for fertilizer (flat log is dangerous — no rollback)
5. Fertilizer needs longer hold than food (all agree on this, debate is only on duration)

---

## Hard questions to answer first (Devil's Advocate)

1. **Why now?** What makes this the right moment while trading bot and content creation remain unfinished?

2. **Personal or external value?** Is this for personal knowledge only, or does it connect to content creation / teaching / selling? (This determines the investment level justified)

3. **Who controls the farm?** Do you need family/farm manager agreement before designating test plots? Fertilizer test failure on a family farm has relational cost beyond crop loss.

4. **What does fertilizer success look like in numbers?** e.g., "cost per tree reduced 20%+ while bunch weight stays within 10% of baseline" — without this, no outcome can be cleanly evaluated.

5. **Do organic input sources exist locally at low/zero cost?** Rice husks, palm empty fruit bunches, manure? If all inputs must be purchased, the cost-reduction premise may not hold.
