# Optimist: /system-review as Compounding Intelligence Loop

Build `/system-review` as standalone command. Data plumbing exists already — healthcheck, analyst, paper-survey each produce signal independently. Missing piece is synthesis prompt. 80% prompting, 20% file structure. 1-2 hour build.

**Best case:** Monthly runs surface 2-3 systemic improvements per cycle. System compounds — OUTCOMES.md accumulates, each run more accurate. By month 6, roadmap becomes primary driver of LTD-OS evolution.

**Realistic case:** Catches broken scripts faster, flags 1 cost inefficiency/month, keeps command list from growing stale. Saves 30-60 min/month overhead.

**Output format:** 3-tier roadmap (fix-now / improve-this-week / backlog), max 10 items. Cap 600 words.

**Manual first.** After 3 successful runs evaluate scheduling. Extending /weekly-calibration = context mix (trading scope). Extending /analyst = cost-lens mismatch. Clean new command keeps concerns separated.

**Risks:**
- Synthesis too generic → seed with real healthcheck outputs first
- Cost creep if scheduled too fast → start manual, schedule only after 3 runs
- Bloat → hard limit 600 words, 3-tier structure locked
