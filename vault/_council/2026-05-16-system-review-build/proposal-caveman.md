# Caveman: Use What Already Exists

UNEASY. Not danger. Uneasy. 30+ commands already. Adding new command to fix problem caused by too many things — smells wrong.

**Real pain?** Yes. 14 papers sit unused. That is real rot. But answer is not new command.

**Cost (primal):** 2 hours build. Brain space to remember it exists. One more thing that can break.

**Simplest version:** Add one section to `/analyst`. "Architecture gaps." 20 minutes, not 2 hours. Same trigger, same habit.

**Danger signals:**
- Scheduled auto-run = fire alarm in empty cave. One-person system, no one to hear it.
- 14 papers not implemented = food on table we're not eating. New fork doesn't fix hunger.
- New command = new habit. Habits hard. Caveman forgets.

**Gut signal on auto-schedule:** DANGER. Smoke detector beeping in empty room.

**Kill condition:** If cannot explain as "I run this, it tells me exactly what to fix next, I fix it" in one real session → do not build. Add to analyst instead and test 2 weeks first.
