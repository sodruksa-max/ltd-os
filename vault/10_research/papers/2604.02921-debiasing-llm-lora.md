# Debiasing LLM ด้วย LoRA

## สรุปสั้น
Fine-tune Qwen3-32B ด้วย LoRA บน financial data → ขจัด extrapolation bias และ overreaction bias ได้สำเร็จ OOS bias coefficient ลดจาก -0.456 เหลือ -0.027 (ไม่ significant) cost "a few hundred dollars"

## Biases ที่แก้
1. **Extrapolation bias**: ให้น้ำหนัก recent trend มากเกินไป → project ไปในอนาคต
2. **Overreaction**: update forecast รุนแรงเกินไปตาม recent data

## ผล (Qwen3-32B)

| Metric | Before LoRA | After LoRA |
|--------|-------------|------------|
| Overreaction coeff (AR1) | -0.456 ** | -0.027 ns |
| Recency bias (stock) | +0.394 *** | -0.120 *** |
| Interpretation | Trend-chasing | Mean-reversion aware |

## Training Details (เท่าที่ paper ระบุ)
- Model: Qwen3-32B (ถ้ามี GPU เล็กกว่า → ลองกับ Qwen2.5-7B)
- LoRA rank r = 8-16 (low-rank matrices A, B)
- Freeze base weights → train เฉพาะ A, B (<1% of params)
- Dataset: ~30,000 obs per persistence level
- Train/Val/Test split: 2001-2011 / 2012-2015 / 2016-2024
- Early stopping บน validation
- Temperature = 0 (greedy decoding)
- Cost: few hundred USD บน cloud GPU

## LoRA Formula
```
h = W₀x + BAx
```
- W₀ = frozen base weights
- B (d×r), A (r×k) = trainable low-rank matrices
- ΔW = BA (starts at zero เพราะ B init = 0)

## 7-Phase Implementation Guide

```
Phase 1: Setup
  → model: Qwen2.5-7B (ถ้า local) หรือ Qwen3-32B (cloud)
  → libs: transformers, peft, accelerate, vLLM

Phase 2: Baseline bias measurement
  → prompt model ด้วย BTC price history sequences
  → regress predictions บน lagged returns
  → บันทึก bias coefficients

Phase 3: Training data
  → prompt-response pairs: input=price history, target=rational benchmark
  → rational benchmark = mean-reverting model forecast
  → อย่า include firm names/dates (prevent lookahead from pretraining)

Phase 4: LoRA attachment
  → r = 8 ถ้า VRAM น้อย, r = 16 ถ้าพอ
  → target_modules = ["q_proj", "v_proj"]

Phase 5: Training
  → AdamW optimizer, lr = 1e-4 to 5e-4
  → early stopping บน validation correlation
  → save best checkpoint

Phase 6: Merge + deploy
  → W' = W₀ + BA → merge เป็น single model
  → ไม่มี overhead ตอน inference

Phase 7: Validate
  → run test set (held-out entirely)
  → bias coefficients ควร → 0 หรือ reverse sign
```

## Practical สำหรับ crypto bot
- Fine-tune model เฉพาะ financial prediction layer ไม่ใช่ทั้งหมด
- Training data: BTC daily returns + "rational" target = หา mean-reverting baseline ก่อน
- หลัง fine-tune → รัน PolySwarm (2604.03888) ด้วย debiased models
- ร่วมกับ prompt rules จาก 2604.18373 → double protection จาก bias
