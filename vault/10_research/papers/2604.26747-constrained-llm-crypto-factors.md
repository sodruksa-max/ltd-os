# Constrained LLM Agents for Crypto Factor Discovery

## สรุปสั้น
ให้ LLM เป็น "นักวิจัย" ที่วนหา trading factor ในตลาด crypto โดยจำกัดด้วย DSL (Domain Specific Language) เพื่อกัน hallucination และ lookahead bias ผลคือ Sharpe 2.4 OOS บน daily crypto data

## Pipeline
```
LLM อ่าน experiment log
  ↓ เสนอ hypothesis เป็น DSL recipe
  ↓ Deterministic engine ประเมิน IC, t-stat, Sharpe
  ↓ บันทึกผลใน append-only log
  ↓ วน 5 รอบ
  ↓ Ridge regression รวม factors ที่ผ่าน
  ↓ Test OOS
```

## Constraint System (สำคัญมาก — ป้องกัน AI มั่ว)
- **Fixed DSL grammar**: ใช้ได้แค่ cross-sectional rank, time-series transform, nonlinear functions ที่กำหนดไว้ — ห้ามเขียน Python ตรงๆ
- **Deterministic gates**: IC threshold + t-stat threshold ตายตัว — LLM แก้ไม่ได้
- **Append-only log**: ทุก hypothesis + ผล บันทึกหมด ย้อนดูได้
- **Separated roles**: LLM แค่เสนอ direction — computation engine คำนวณเอง

## Dataset และผล
- Data: CoinMarketCap daily, Jan 2020–Dec 2025 (filtered >180 days + min liquidity)
- Train: 2020–2022 | Val: 2023 | OOS: 2024–2026
- **25 factors generated → Pure OOS Sharpe +2.412**
- L/S portfolio: 44.55% annualized return, Sharpe 1.55 (หลังหัก 5bps cost)
- Alpha อยู่ใน small-cap tokens (liquidity-scarce) — large-cap ไม่ work

## การนำไปใช้กับ bot
- ใช้ local LLM (Ollama / Llama) แทน GPT เพื่อ privacy + cost
- เก็บ experiment log ใน SQLite หรือ Parquet
- DSL implement เป็น Python AST validator (whitelist functions)
- ดึงข้อมูลด้วย ccxt
- รัน backtest ด้วย vectorbt หรือ Backtrader

## ข้อควรระวัง
- Alpha อยู่ใน small-cap → ระวัง slippage สูง
- Market-cap weighting พัง — ต้องใช้ equal-weight หรือ signal-weight
