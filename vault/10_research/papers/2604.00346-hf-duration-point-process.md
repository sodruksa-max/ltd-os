# Self-Exciting Point Process: Predict เมื่อตลาดจะขยับ

## สรุปสั้น
Model ความถี่ของ trade arrival ด้วย self-exciting process ที่ประมาณ expected time ถึง trade ถัดไปได้ เอาไปตัดสินใจว่าตอนนี้ควร "รอ" หรือ "act เลย"

## Core Model

### State Update
```
Λₙ = μ + (Λₙ₋₁ − μ + α) · exp(−β · τₙ)
```
- Λₙ = latent intensity (ความถี่โดยประมาณ)
- μ = baseline intensity
- α = self-excitation (แต่ละ trade ดัน intensity ขึ้น)
- β = decay rate (intensity ลดลงตามเวลา)
- τₙ = time since last trade

### Expected Next Arrival Time
```
E[τₙ₊₁ | Fₙ] = ∫₀^∞ s · fₑ(Φ(s, Λₙ)) · Ψ(s, Λₙ) ds
```

### Stability Condition
```
α < β · μₑ  (ต้องการเพื่อ stationarity)
```

## ผล (AAPL Dec 2022)
- Mean duration: 0.48 วินาที
- Skewness 5.75, kurtosis 61.46 (heavy-tailed มาก)
- SE-Burr model ดีที่สุด (best balance accuracy + distributional fit)

## Decision Framework สำหรับ bot

```python
class TradingTimingModel:
    def __init__(self, mu=1.0, alpha=0.5, beta=2.0):
        self.mu = mu
        self.alpha = alpha
        self.beta = beta
        self.Lambda = mu  # current intensity

    def update(self, tau_n):
        """อัปเดต intensity หลังได้รับ trade ใหม่"""
        self.Lambda = self.mu + (self.Lambda - self.mu + self.alpha) * np.exp(-self.beta * tau_n)

    def expected_next_duration(self):
        """ประมาณเวลาถึง trade ถัดไป"""
        return 1.0 / self.Lambda  # approximation

    def should_act(self, threshold_seconds=30):
        """
        E[next] สั้น = ตลาด active = act เลย
        E[next] นาน = ตลาด quiet = รอ
        """
        expected = self.expected_next_duration()
        return expected < threshold_seconds

    def momentum_strength(self):
        """α/β ratio สูง = momentum persist = favor trend"""
        return self.alpha / self.beta
```

## Decision Rules
```
E[τₙ₊₁] < 100ms  → ตลาด clustered → act decisively
E[τₙ₊₁] > 30s   → ตลาด quiet → รอ หรือลด size
α/β ratio สูง    → momentum ต่อเนื่อง → trend-following mode
```

## การนำไปใช้กับ crypto bot
- Re-estimate (μ, α, β) ทุก ~45 นาที (rolling 5,000 trades)
- ใช้ร่วมกับ LOB regime detector (2604.20949) — สองตัวยืนยันกัน
- ถ้า E[τ] สั้น + regime alert → ระวังเข้าตอนนั้น
- ถ้า E[τ] นาน → ตลาด consolidating → รอ breakout

## ข้อควรระวัง
- Model assume no exogenous shocks — ถ้ามีข่าวใหญ่ต้อง override manually
- ทดสอบกับ AAPL (equity) ไม่ใช่ crypto — BTC มี spikier intensity
- ต้องการ tick-level data (not just 15-min OHLCV)
