# A2A Blockchain Payments

## สรุปสั้น
Systematization of Knowledge (SoK) เรื่อง agent-to-agent payment infrastructure บน blockchain โดย agents ทำธุรกรรมเองโดยไม่ต้องมี human approve ทุกครั้ง มีการพูดถึง OpenClaw และ X402 เป็น early prototypes

## 4-Stage Lifecycle

```
1. Discovery   → agent หา counterparty / service ที่ต้องการ
2. Authorization → enforce spend limit + policy constraint
3. Execution   → submit transaction (account abstraction)
4. Accounting  → บันทึก verifiable on-chain record
```

## Frameworks ที่กล่าวถึง
- **OpenClaw**: early prototype สำหรับ agent-centric ecosystem
- **Moltbook**: อีก prototype ในแนวเดียวกัน
- **X402**: open standard สำหรับ internet-native machine-to-machine payment

## การนำไปใช้กับ crypto bot (future phase)
- ใช้ A2A infrastructure ให้ Analysis Agent สั่ง Execution Agent โดยตรงผ่าน on-chain proof
- Authorization layer: ตั้ง spend limit ต่อ transaction, ต่อวัน
- Accounting: ทุก trade มี on-chain evidence ตรวจสอบได้

## ข้อควรระวัง
- Paper เน้น infrastructure ไม่ใช่ trading strategy
- **ยังเป็น early-stage** — production deployment ยังมี open challenges เรื่อง payment-service coupling
- เหมาะ reference เมื่อ bot ถึง phase ที่ต้องการ autonomous multi-agent coordination จริงๆ
