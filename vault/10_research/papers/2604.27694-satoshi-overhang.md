# Satoshi Overhang: Bear Case มีขอบเขต

## สรุปสั้น
1.148 ล้าน BTC (~5.7% ของ supply) ที่ Satoshi mine ในปี 2009-2010 ยังไม่เคยขยับ 16 ปี paper นี้ argue ว่าถ้า Satoshi ขาย market depth ปัจจุบันรับได้โดยราคาลงแค่ 5-25% ไม่ใช่ crash 90%

## Scenario Analysis

| Scenario | Elasticity | Timeframe | Cumulative Impact |
|----------|-----------|-----------|-------------------|
| Conservative | 1.5 | 12 ปี | -5% ถึง -6% |
| Base Case | 0.7 | 10 ปี | -11% ถึง -12% |
| Aggressive | 0.3 | 5 ปี | -23% ถึง -25% |

**Empirical anchors:**
- Silk Road auction (disciplined): 2–5% per tranche
- German police 2024 dump (leverage contaminated): 15–20%

```python
# Probability distribution ของ outcomes

    "estate_unlock": 0.05,       # ทายาทขาย
}

# Mechanical impact cap (ใช้ 15% เป็น conservative upper bound)
SATOSHI_MAX_IMPACT = 0.15

def satoshi_risk_adjustment(position_size_usd, satoshi_alert=False):
    if satoshi_alert:
        # ลด long leverage ถ้าตรวจพบ Satoshi address ขยับ
        return position_size_usd * 0.5

```

## Signal Hierarchy (เรียงตามความน่าเชื่อถือ)
1. Cryptographically-verifiable death proof → ลบ risk ทิ้ง
2. OP_RETURN burn transaction → bullish ถาวร
3. Custody transfer (cold → exchange) → reduce position
4. Public announcement → monitor
5. Address active (first move in 16 ปี) → reduce long leverage ทันที

## Volatility Pattern ถ้าเกิดจริง
- **Short-term**: overshoot 10-15% ใน hours-to-days
- **Long-term**: mean reversion (absorption โดย patient buyer)
- Treat เหมือน high-impact macro event ไม่ใช่ existential threat

## การนำไปใช้กับ bot
- ตั้ง alert บน Satoshi address (blockchain monitor)
- ถ้า alert → cap position size 50%, เพิ่ม stop loss หน้า
- Tail hedge cost ≤ 2-3% (ไม่ใช่ 5-10% สำหรับ existential scenario)
