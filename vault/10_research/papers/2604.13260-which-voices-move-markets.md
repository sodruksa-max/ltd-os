# Source-Weighted News Sentiment

## สรุปสั้น
ใน earnings call: analyst questions มีน้ำหนัก 49% ทั้งที่พูดแค่ 20% ของ sentence — เพราะ analysts พูดโดยไม่ได้เตรียม (genuine) ส่วน CEO พูด 51% แต่มีน้ำหนักแค่ 16% OOS IC 0.142, monthly alpha 2.03%

## Optimal Weights ตาม Source

| Source | % of sentences | Optimal weight |
|--------|---------------|----------------|
| Analyst | 20.2% | **48.8%** |
| CFO | — | 29.5% |
| Executive | 51.0% | 15.9% |
| Other | — | 5.8% |

น้ำหนัก analyst สูงเพราะ spontaneous (genuine skepticism) ส่วน executive เตรียมมาพูด (strategic tone management)

## Methodology
1. คำนวณ Spearman IC ระหว่าง sentiment ของแต่ละ speaker category กับ returns ถัดมา
2. Weight allocation ∝ predictive power ของแต่ละ category
3. Freeze weights รายไตรมาส (ห้าม dynamic update เพื่อหลีกเลี่ยง lookahead)
4. ใช้ FinBERT (ไม่ใช่ dictionary-based) — FinBERT subsume dictionary อย่างสมบูรณ์

## Crypto Adaptation

**Paper ทดสอบกับ equity earnings call เท่านั้น** — ต้อง adapt:

```python
# Crypto source categories (แทน speaker roles)
CRYPTO_SOURCE_WEIGHTS = {
    "core_dev_announcement": 0.40,   # เทียบกับ analyst — most credible
    "institutional_report": 0.30,    # เทียบกับ CFO
    "exchange_official": 0.20,       # เทียบกับ executive
    "retail_social_media": 0.10,     # noise มากที่สุด
}

def weighted_sentiment_score(news_items: list) -> float:
    total_weight = 0
    weighted_sum = 0
    for item in news_items:
        source_type = classify_source(item['source'])  # FinBERT หรือ rule-based
        weight = CRYPTO_SOURCE_WEIGHTS.get(source_type, 0.10)
        sentiment = get_sentiment(item['text'])  # FinBERT -1 to +1
        weighted_sum += weight * sentiment
        total_weight += weight
    return weighted_sum / (total_weight + 1e-10)
```

## การนำไปใช้กับ bot
- ใช้ FinBERT (หรือ FinBERT-crypto variant) ไม่ใช่ VADER/TextBlob
- Recalibrate weights ทุก 3 เดือน (freeze อย่าปรับ real-time)
- Core dev + institutional > exchange official > social media เสมอ
- ถ้า core dev sentiment ลบ → bearish signal แม้ social bullish

## ข้อควรระวัง
- Crypto ไม่มี standardized disclosure → source verification ยาก
- Signal decay เร็วกว่า equity (ชั่วโมง ไม่ใช่วัน)
- ต้องการ historical data per source type เพื่อ calibrate IC ก่อน deploy
