# Self-Driving Portfolio: 50+ Agent Architecture

## สรุปสั้น
~50 specialized agents ทำงาน sequential ตั้งแต่ market analysis → portfolio construction (20+ methods) → critique-and-vote → meta-agent monitor performance ตาม IPS (Investment Policy Statement)

## Agent Layers

```
Layer 1: Market Assumption Agents
  → macro, sector, factor analysts

Layer 2: Portfolio Construction Agents (20+ competing methods)
  → mean-variance, risk parity, factor model, etc.

Layer 3: Critique-and-Vote
  → agents ประเมิน + โหวต output ของ layer 2

Layer 4: Meta-Agent
  → compare past forecasts vs realized returns
  → update agent code + prompts ตาม performance
```

## Critique-and-Vote Mechanism
- หลาย agents วิเคราะห์ portfolio ที่ layer 2 สร้าง
- Vote บน quality + consistency กับ IPS
- Meta-agent เรียนรู้จาก forecast error → refine agents อัตโนมัติ

## ผล
- Paper ไม่มีตัวเลข quantitative (abstract เท่านั้น)
- Focus ที่ architecture ไม่ใช่ performance metrics

## Crypto Blueprint (Minimal Version)

```
Layer 1: 3 analysts (macro, on-chain, technical)
Layer 2: 3 strategies (momentum, mean-reversion, breakout)
Layer 3: 1 critic agent → vote ว่า strategy ไหน fit regime
Layer 4: 1 meta-agent → track PnL per strategy, reweight รายสัปดาห์
```

## ข้อควรระวัง
- Original paper เน้น institutional (มี IPS constraints) — crypto ต้อง adapt governance rules
- 50 agents = expensive มาก local LLM — ลดเหลือ 5-10 สำหรับ practical use
