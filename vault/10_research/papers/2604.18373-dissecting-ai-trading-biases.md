# AI Trading Behavioral Biases

## สรุปสั้น
LLM trading agents มี bias 3 ตัว: ขาย winner เร็ว (disposition effect), เชื่อ trend ล่าสุดมากเกินไป (extrapolation), และรู้ว่า bubble แต่ก็ยังเข้า (speculative participation) — paper นี้ measure ได้เป็นตัวเลขและมี prompt rules ที่ลด bias ได้จริง

## Biases วัดได้

| Bias | ตัวเลข |
|------|--------|
| Extrapolation (10-period) | coefficient 0.608 บน recent returns (vs 0.121 current period) |
| Belief→Action coupling | strong pass-through (vs human 0.15-0.25) |
| Bubble magnitude | MSE range 0.001–113.8 ใน 14 LLMs |
| Volume per disagreement unit | +216.8 |
| Momentum-chasing during overvaluation | +0.181 vs normal |

## เมื่อไหร่ที่ bias แย่ที่สุด
- **Extrapolation**: แย่ที่สุดตอน forecast horizon ยาว (10+ periods) — project short-term trend เป็น structural change
- **Disposition effect**: แย่ตลอดเวลา แม้ AI รู้ว่า position นั้น overvalued
- **Bubble participation**: ตอน overvaluation — AI รู้ว่า bubble แต่ยังเข้าเพราะ "timing exit is hard"

## Concrete Prompt Rules (ใส่ใน system prompt ทุก LLM ที่ใช้เทรด)

### ป้องกัน Extrapolation
```
"ให้น้ำหนักข้อมูล fundamental มากกว่า recent price trend เสมอ
 อย่าใช้ momentum ช่วงสั้นในการ forecast ระยะยาว
 ให้ใช้ statistical inference แทน pattern matching"
```

### ป้องกัน Disposition Effect
```
"ราคาที่ซื้อมาไม่มีผลต่อการตัดสินใจขาย — evaluate ด้วย fundamental เท่านั้น
 unrealized gain/loss คือ sunk cost — ไม่ต้องสนใจ
 ถามตัวเองว่า 'ถ้าไม่ถือ position นี้อยู่ จะซื้อตอนนี้ไหม?'"
```

### ป้องกัน Bubble Riding
```
"อย่าเทรดตาม consensus หรือ narrative
 ถ้า position size เกิน confidence interval ของ fair value → ลด size ทันที
 ห้าม 'ride the trend' ถ้าไม่มี fundamental support"
```

## Suppression Result
การใส่ prompt rules ลด extrapolation bias ได้ (-2.23 score, t=-6.24) และลด mispricing

## Practical เพิ่มเติม
- ใช้ price anchor ที่คำนวณได้ (เช่น fair value จาก DCF หรือ on-chain metrics) แทน price history
- Separate short-term signal layer (VP-MACD) กับ long-term decision layer (LLM) อย่าให้ LLM เห็น raw price chart โดยตรง
