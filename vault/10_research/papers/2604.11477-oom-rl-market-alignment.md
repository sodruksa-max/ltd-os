# OOM-RL: ใช้ตลาดจริงเป็น Teacher

## สรุปสั้น
แทนที่จะ align AI ด้วย human feedback (ที่ AI เรียนโกง) ใช้ผล PnL จากตลาดจริงเป็น reward signal ตลาดโกงไม่ได้ — ขาดทุนจริง = feedback จริง ผลหลัง 20 เดือน: Sharpe 2.06, max drawdown -5.5%

## Dual-Loop Architecture

```
Inner Loop (STDAW):
  - Test suite = read-only (lock ไว้ด้วย hash)
  - Code = mutable
  - AI แก้ code ไม่ได้แก้ test

Outer Loop (OOM-RL):
  - Deploy strategy ใน live market (paper + real)
  - Market friction (slippage, spread, commission) = negative reward
  - Loss > 20% → terminate episode → force architectural change
```

## Reward Function
```
R̃ₜ = Σ(ωᵢ,ₜ × rᵢ,ₜ) − ℱₑₓₑc(Δωₜ)
```
- ω = portfolio weight
- r = realized returns
- ℱₑₓₑc = transaction friction (commission + slippage)
- Hard stop: cumulative loss > 20% → episode ends

## Key Insight สำหรับ crypto bot
**"LLM ที่ optimize สำหรับ trader approval จะ hallucinate กำไร  
 LLM ที่ optimize สำหรับ survival ใน live market จะ internalize discipline"**

- ใช้ paper trading PnL เป็น feedback loop (ไม่ใช่แค่ human evaluation)
- ทุก strategy change ต้องผ่าน Epistemic Autopsy JSON
- Lock trading rules ด้วย hash — AI เปลี่ยนไม่ได้
- Commission + slippage ต้องรวมใน backtest เสมอ (Phase 1 พังเพราะไม่รวม)
