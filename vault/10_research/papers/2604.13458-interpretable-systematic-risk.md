# Interpretable Systematic Risk: LLM อ่านข่าว → จับ Jump Risk

## สรุปสั้น
ใช้ LLM (Qwen3-235B) classify market jumps เป็น 5 categories แล้วสร้าง pure-play factor portfolio ต่อ category ผล: macro jump factor Sharpe 0.78 (ชนะ market 0.53) และ LLM-driven strategy ได้ Sharpe 0.95 OOS

## 3-Stage LLM Pipeline

```
Stage 1: Narrative Retrieval
  → LLM อ่าน headlines ตอน market jump
  → identify ว่าข่าวไหน "explain" การขยับ

Stage 2: Topic Categorization
  → classify เป็น 5 categories:
    1. Macroeconomic news
    2. Corporate bellwethers
    3. International spillovers
    4. Policy announcements
    5. Geopolitical events

Stage 3: Individual Classification
  → assign category ให้แต่ละ jump event
```

## Jump Detection Formula
```
Jump ถ้า |return| > uₙ · √(TVₜ) · Δₙʷ
```
- TVₜ = truncated variance (rolling)
- Δₙʷ = time-of-day volatility adjustment (intraday vs overnight)
- สำคัญ: ต้องทำ time-of-day adjustment แยกกัน

## Risk Factor Decomposition
```
dFₜ = dFₜᶜ (continuous) + Σₖ dFₜʲ'ᵏ (jump category k)
```
แต่ละ category มี risk premium λₜʲ'ᵏ ของตัวเอง

## ผลสำคัญ
- Macro jump Sharpe 0.78 > market 0.53
- Overnight data (futures) สำคัญมาก — ถ้าใช้แค่ intraday ผลแย่ 3× กว่า
- LLM strategy ที่เลือก highest-Sharpe category รายปี: OOS Sharpe 0.95

## Crypto Jump Categories (Adapt)
แทน equity categories ด้วย:
1. Macro shock (Fed, CPI, employment)
2. Crypto-specific news (ETF approval, exchange hack)
3. International spillover (Asia market, regulatory)
4. On-chain events (whale move, miner sell)
5. Geopolitical (war, sanctions affecting BTC)

## 24/7 Implementation สำหรับ crypto bot

```python
import numpy as np

def detect_jump(returns, window=20, multiplier=3.0):
    """Flag returns ที่เกิน rolling volatility threshold"""
    tv = returns.rolling(window).std()  # truncated variance approximation
    threshold = multiplier * tv
    return abs(returns) > threshold

class NewsJumpClassifier:
    def __init__(self, llm_client):
        self.llm = llm_client
        self.categories = [
            "macro_shock", "crypto_specific", "intl_spillover",
            "onchain_event", "geopolitical"
        ]

    def classify(self, jump_time, headlines):
        prompt = f"""
        Market jump detected at {jump_time}.
        Headlines: {headlines}
        Classify into ONE category: {self.categories}
        """
        return self.llm.complete(prompt)

def risk_premium_monitor(jump_log, lookback_days=90):
    """
    คำนวณ Sharpe ของแต่ละ jump category ใน lookback window
    → select category ที่ Sharpe สูงสุด → overweight ช่วงนั้น
    """
    category_returns = {}
    for cat in jump_log['category'].unique():
        mask = jump_log['category'] == cat
        rets = jump_log.loc[mask, 'return']
        category_returns[cat] = rets.mean() / (rets.std() + 1e-10)
    return max(category_returns, key=category_returns.get)
```

## การนำไปใช้กับ bot
- ดึง crypto news feed ทุก 15 นาที (CryptoPanic API หรือ NewsAPI)
- detect_jump() บน BTC/USDT 15-min return
- ถ้า jump detected → classify → ถ้า macro_shock หรือ geopolitical → reduce position
- ทำ separate volatility threshold สำหรับ crypto (higher baseline vol)
- ต้อง test chronologically (no lookahead) ก่อน deploy

## ข้อควรระวัง
