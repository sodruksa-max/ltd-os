# PolySwarm: 50 Personas โหวตทิศทาง BTC

## สรุปสั้น
ใช้ 50 LLM personas (macro economist, technical analyst, contrarian ฯลฯ) sample 25 ตัวต่อรอบ แต่ละตัวให้ probability + confidence แล้ว aggregate ด้วย 2-stage Bayesian combination ชนะ single-model baseline ใน calibration

## Architecture
```
50 personas pool (macro, technical, contrarian, momentum, etc.)
  ↓ sample 25 ต่อรอบ (without replacement)
  ↓ parallel independent predictions
  ↓ Stage 1: confidence-weighted swarm consensus
  ↓ Stage 2: Bayesian combination กับ market price
  ↓ Final signal: BUY / SKIP
```

## Formulas

### Stage 1 — Swarm Consensus
```
p_swarm = Σ(w_i × p_i) / Σ(w_i)
```
- w_i = confidence ของแต่ละ agent
- p_i = probability UP ของแต่ละ agent

### Stage 2 — Market Integration
```
p_combined = 0.70 × p_swarm + 0.30 × p_market
```
Swarm dominates (70%) แต่ยัง incorporate market information (30%)

## Python Implementation

```python
import asyncio
from dataclasses import dataclass
from typing import List

@dataclass
class AgentPrediction:
    persona: str
    confidence: float  # 0-1
    probability: float  # P(price UP)

class PolySwarmBTC:
    PERSONAS = [
        "macro_economist", "technical_analyst", "contrarian",
        "momentum_trader", "on_chain_analyst", "volatility_specialist"
    ]

    def confidence_weighted_swarm(self, preds: List[AgentPrediction]) -> float:
        total_w = sum(p.confidence for p in preds)
        return sum(p.confidence * p.probability for p in preds) / (total_w + 1e-10)

    def bayesian_combine(self, p_swarm: float, p_market: float) -> float:
        return 0.70 * p_swarm + 0.30 * p_market

    def trade_signal(self, p_final: float, preds: List[AgentPrediction]) -> dict:
        std = (sum((p.probability - p_final)**2 for p in preds) / len(preds)) ** 0.5
        return {
            "direction": "UP" if p_final > 0.5 else "DOWN",
            "probability": p_final,
            "agreement": std,  # ต่ำ = เห็นตรงกัน = signal แข็งแกร่งกว่า
            "signal": "BUY" if p_final > 0.55 and std < 0.30 else "SKIP"
        }
```

## การ adapt สำหรับ crypto bot
- ลดจาก 50 เป็น **5-10 personas** สำหรับ local LLM (ประหยัด compute)
- Personas ที่เหมาะกับ crypto: macro, on-chain, sentiment, technical, contrarian
- Market price (p_market) ใช้ funding rate หรือ options implied vol แทน prediction market
- ถ้า agreement (std) สูง → signal อ่อน → reduce position size

## ข้อควรระวัง
- Paper ทดสอบกับ Polymarket (prediction market) ไม่ใช่ spot crypto trading
- 50 agents ต่อรอบ expensive มาก — ใช้ local model หรือลด sample size
- ระวัง: independent sampling สำคัญมาก ห้ามให้ agents เห็น output กัน (สร้าง groupthink)
