# Thread format (X / Twitter)

## Structure
- **Hook** (post 1): 1-2 sentences. Must earn the scroll-stop.
- **Context** (post 2): why this matters, who this is for
- **Body** (posts 3-N): one idea per post, ordered for flow not thoroughness
- **Payoff** (second-to-last): the insight / the reveal / the numbers
- **CTA** (last): optional — follow for more, link to longform, ask a question

## Length
- 5-12 posts optimal
- Each post: 200-270 chars (leave room for people to quote)
- First post: ≤ 240 chars, no link (link reduces reach)

## Hook patterns (pick one, vary)
- **Contrarian**: "Everyone says X. That's wrong."
- **Numbered**: "3 things about <topic> no one tells you"
- **Story**: "Last week I <specific action>. Here's what I learned."
- **Question**: "Why does <counterintuitive thing> happen?"
- **Stake**: "I spent <time/money> on <thing>. Breakdown."

Avoid:
- "Let me tell you about..."
- "Thread 🧵" (assume people know)
- Vague hooks ("AI is changing everything")

## Rhythm
- Alternate short and longer posts
- Don't have 3+ posts in a row that start the same way
- End each post with tension or payoff — reason to swipe

## Evidence per post
Every claim needs ONE of:
- Number with source
- Specific example
- Quote (attributed, < 15 words)
- Screenshot / image callout

## Thai threads
- Mix Thai + English technical terms (don't translate "hook", "CTA", ticker symbols)
- Short sentences carry better in Thai
- Avoid directly translating English thread patterns — rhythm is different

## Don't do
- "1/" numbering (X auto-threads now)
- Cliffhangers that don't deliver in next post
- Same-structure posts repeating
- Self-promotion before value

## Example skeleton
```
[1/ hook — 240 chars]
[2/ context — why now, who for]
[3/ point A — specific]
[4/ evidence for A]
[5/ point B — specific]
[6/ evidence for B]
[7/ counter-point / nuance]
[8/ synthesis — the insight]
[9/ CTA or payoff]
```
